# LeetCode 70「爬楼梯」动画设计

## 目标

新增一个零依赖的交互式动画页面，通过连续的三个章节解释递归、记忆化搜索和滚动动态规划之间的演进关系。用户应能看懂重复子问题如何产生、缓存如何消除重复计算，以及 DP 如何把额外空间压缩到 O(1)。

## 范围

- 新增 `problems/70-climbing-stairs.html`。
- 更新 `README.md` 的题目清单。
- 不修改首页、清单生成脚本或部署流程。
- 不处理与本功能无关的文件，包括未跟踪的 `dify-self-host-guide.md`。

## 页面结构

页面沿用现有题目的原生 HTML、内联 CSS 和内联 JavaScript 结构，不引入第三方依赖。

内容从上到下包括：

1. 难度标签、LeetCode 原题链接、标题和算法简介。
2. 输入区：整数 `n`、载入按钮和常见示例。
3. 图例和复杂度对比摘要。
4. 动画控制区：上一步、下一步、自动播放、重置和总进度。
5. 引导式三章节动画区：
   - 递归：暴露重复子问题。
   - 记忆化：展示缓存写入和命中。
   - 滚动 DP：展示状态转移和空间压缩。
6. 当前状态说明和最终答案。
7. Java、Python 参考代码，覆盖三种解法。

统一时间线贯穿三个章节。当前章节随步骤自动切换，用户不需要手动管理三个彼此独立的动画状态。

## 输入与错误处理

- 只接受 `1–10` 的整数。
- 选择该范围是为了完整展示递归调用树，同时避免节点数失控。
- 非整数、空输入、`0`、负数以及大于 `10` 的值均显示明确的行内错误。
- 非法输入不会被静默截断、四舍五入或替换，也不会破坏当前有效动画。
- 载入新输入、重置页面或手动切换步骤时，应停止正在运行的自动播放。

## 动画模型

载入合法输入后，JavaScript 一次性构建不可变的步骤快照。渲染函数只读取当前快照，因此“上一步”能准确恢复此前状态，无需逆向执行算法。

每个快照至少包含：

- 所属章节和章节内阶段；
- 当前处理的节点或楼层；
- 已知返回值、缓存或滚动变量；
- 调用次数和复杂度提示；
- 面向学习者的状态说明；
- 是否已完成及最终答案。

### 第一章：朴素递归

- 从 `f(n)` 开始逐步展开完整调用树。
- 使用基例 `f(1)=1`、`f(2)=2`。
- 高亮当前调用路径、已返回节点和重复出现的子问题。
- 返回时在节点上显示计算结果。
- 记录总调用次数，直观说明指数级重复计算。

### 第二章：记忆化搜索

- 使用相同的递推关系和基例，便于与第一章直接比较。
- 区分首次计算、写入缓存和缓存命中。
- 持续显示当前缓存内容与调用次数。
- 最终说明时间复杂度降为 O(n)，额外空间为 O(n)。

### 第三章：滚动动态规划

- 用楼梯格表示从第 1 阶到第 `n` 阶的状态。
- 每步高亮 `f(i-2)`、`f(i-1)` 和当前 `f(i)`，展示 `f(i)=f(i-1)+f(i-2)`。
- 先建立递推直觉，再明确只需保留前两个状态。
- 显示滚动变量的更新前后值。
- 最终说明时间复杂度 O(n)、额外空间 O(1)。

三个章节始终产生相同答案。页面持续展示方法名称、调用或迭代次数、时间复杂度、空间复杂度和最终结果，帮助用户建立对比。

## 视觉与响应式设计

- 复用仓库现有的浅灰背景、白色卡片、靛蓝主色和深色状态栏。
- 递归、记忆化和 DP 使用可区分且带文字图例的配色，避免仅依赖颜色表达状态。
- 递归树在窄屏上允许横向滚动，不压缩到无法阅读。
- 其余模块在窄屏下改为单列，控制按钮允许换行。
- 首步禁用“上一步”，末步禁用“下一步”；自动播放按钮能明确显示播放或暂停状态。

## 验证

- `n=1` 返回 `1`。
- `n=2` 返回 `2`。
- `n=5` 返回 `8`。
- `n=10` 返回 `89`。
- 三个章节对同一输入给出相同结果。

### 交互

- 上一步、下一步、自动播放和重置在三个章节边界处保持连续。
- 自动播放到末步后自行停止。
- 载入新输入时停止旧计时器并回到首步。
- 非法输入 `0`、`11`、小数、空值和文本均显示错误，且保留当前有效动画。

### 页面与集成

- 桌面和窄屏布局均可读，递归树可滚动。
- Java 和 Python 参考代码与动画采用相同的基例和递推关系。
- 运行 `node scripts/gen-manifest.js` 后，生成清单包含题号 70、正确标题和“简单”难度。
- 检查 Git 状态，确保 `problems.json` 及无关未跟踪文件未进入提交。
