215. 数组中的第 K 个最大元素 · 快速选择（三路划分）动画

升序下第 K 大在下标 n − k。随机基准 + 三路划分把区间切成「小于 / 等于 / 大于」三块，只往含目标的一侧继续。三路划分让大量重复元素也能一次归位，期望 O(n)（这也是能通过 LeetCode 全部用例的关键）。

// Java · 随机基准 + 三路划分 · 期望 O(N)
import java.util.Random;
class Solution {
    private final Random rnd = new Random();
    public int findKthLargest(int[] a, int k) {
        int n = a.length, target = n - k;    // 升序第 target 个 = 第 k 大
        int lo = 0, hi = n - 1;
        while (true) {
            if (lo == hi) return a[lo];
            swap(a, lo, lo + rnd.nextInt(hi - lo + 1)); // 随机基准放到区间首
            int pivot = a[lo], lt = lo, gt = hi, i = lo + 1;
            while (i <= gt) {                // 荷兰国旗三路划分
                if (a[i] < pivot)      swap(a, lt++, i++);
                else if (a[i] > pivot) swap(a, i, gt--);
                else                   i++;
            }
            // [lo,lt-1] < pivot；[lt,gt] = pivot；[gt+1,hi] > pivot
            if (target < lt)      hi = lt - 1;   // 目标在小于区
            else if (target > gt) lo = gt + 1;   // 目标在大于区
            else return pivot;                   // 目标落在等于区
        }
    }
    private void swap(int[] a, int x, int y){ int t=a[x]; a[x]=a[y]; a[y]=t; }
}
# Python · 随机基准 + 三路划分
import random
class Solution:
    def findKthLargest(self, a: List[int], k: int) -> int:
        target, lo, hi = len(a) - k, 0, len(a) - 1
        while True:
            if lo == hi: return a[lo]
            r = random.randint(lo, hi)
            a[lo], a[r] = a[r], a[lo]                 # 随机基准
            pivot, lt, gt, i = a[lo], lo, hi, lo + 1
            while i <= gt:                            # 三路划分
                if a[i] < pivot:
                    a[lt], a[i] = a[i], a[lt]; lt += 1; i += 1
                elif a[i] > pivot:
                    a[i], a[gt] = a[gt], a[i]; gt -= 1
                else:
                    i += 1
            if target < lt:   hi = lt - 1
            elif target > gt: lo = gt + 1
            else: return pivot
⏱ 复杂度说明
快速选择每趟只递归一侧，若基准把区间大致对半分，规模按 n + n/2 + n/4 + … ≈ 2n 收缩，得到期望 O(n)。
两个关键优化缺一不可：①随机基准——固定取端点遇到有序输入会退化成 O(n²)；②三路划分——把 = 基准的元素一次性全部归位。若只用两路（< / ≥）分区，遇到大量重复元素（如全相等数组）仍是 O(n²)，会在 LeetCode 超时；三路划分把这种情况也降到线性。
空间 O(1)（原地分区）。严格最坏 O(n) 需「中位数的中位数（BFPRT）」，但常数大，实践中随机基准 + 三路划分足矣。
